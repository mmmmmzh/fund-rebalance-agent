# 隐私与本地数据

## 默认数据流

1. 公开演示数据来自代码生成的合成序列和虚构标识。
2. 用户 CSV 在本机内存中解析，标准化结果写入 `user_data/`。
3. checkpoint、报告、前向信号和日志均写入 `user_data/`。
4. 默认配置不发起网络请求，不调用外部模型，不连接账户。

## CSV 最小化

只保存 `code`、`name`、`current_weight`、`current_value_yuan` 和 `notes`。包含账号、用户名、手机号、邮箱、身份证或银行卡等身份列的 CSV 会被拒绝；未知列、源文件名和上传原文不会保存。

金额仍可能属于敏感财务信息。不要在共享电脑上导入真实金额，不要把 `user_data/`、终端日志或界面截图提交到 GitHub。

## 路径隔离

- 配置只能从 `config/` 或允许的 `user_data/` 根目录加载。
- 运行记录、checkpoint、插件 Skill 和可变 CSV 只能写入 `user_data/`。
- 绝对越界路径、`..` 越界和路径化任务 ID 会被拒绝。
- `FUND_AGENT_USER_DATA_ROOT` 只用于测试或受控宿主环境。

## 删除数据

先停止本地服务，再删除对应的 `user_data/<profile>/` 目录即可清除该配置的持仓、任务、报告和 checkpoint。本项目没有云端副本或后台同步。

## 插件

插件可能引入网络和第三方处理者。安装插件前应单独完成数据授权、个人信息处理、跨境传输、日志和保留周期评估；公开核心不替插件作出这些保证。

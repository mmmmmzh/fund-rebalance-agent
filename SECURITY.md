# 安全策略

## 支持范围

安全修复仅针对最新发布标签和 `main` 分支。公开核心按本地单用户研究工具设计，不应直接暴露到公网，也不应以管理员权限运行。

## 报告漏洞

请使用 GitHub 仓库的 **Security > Advisories > Report a vulnerability** 私下提交：

- 受影响版本与操作系统；
- 最小复现步骤；
- 实际与预期行为；
- 对路径、隐私、代码执行或依赖链的影响。

请不要在公开 Issue 中发布密钥、真实持仓、身份信息或可直接利用的漏洞细节。

## 已有防护

- 配置与写入路径白名单；
- 任务 ID 格式和目录逃逸检查；
- CSV 大小、行数、字段、身份列和数值校验；
- 默认离线且没有账户或外部模型连接；
- 人工确认与不可执行的研究信号；
- 锁定依赖、最小 CI 权限、固定 Action 提交和 `pip-audit`。

## 不支持的部署

公网托管、多租户、共享文件系统、真实交易账户、自动下单和未经审查的插件不属于当前安全模型。
